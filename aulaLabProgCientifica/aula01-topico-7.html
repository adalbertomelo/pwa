3. Gerenciamento de Processos

3.1. Processos e Threads

  • Processo

    Um processo é uma instância de um programa em execução. Ele é composto por:

    • Código Executável: As instruções do programa.
    • Espaço de Endereçamento: Inclui memória para código, dados e pilha.
    • Recursos do Sistema: Como arquivos abertos, dispositivos de I/O e outros recursos alocados.

    Cada processo possui um PCB (Process Control Block) que armazena informações importantes para o gerenciamento do processo, como:

    • Identificação do processo (PID).
    • Estado atual (pronto, executando, bloqueado).
    • Registradores de CPU e contador de programa.
    • Informações de gerenciamento de memória.
    • Informações de segurança e privilégios.

    O sistema operacional utiliza o PCB para rastrear e gerenciar processos, permitindo multitarefa e garantindo isolamento entre processos.

    Exemplo Prático: Quando você abre um editor de texto, o sistema operacional cria um processo que contém o programa do editor, aloca memória e recursos necessários para sua execução.

  • Thread

    Uma thread é a menor unidade de execução que pode ser agendada. As threads compartilham o mesmo espaço de endereçamento e recursos do processo pai, mas possuem seu próprio:

    • Contador de Programa: Para rastrear qual instrução está sendo executada.
    • Conjunto de Registradores: Para armazenar o estado de execução.
    • Pilha: Para chamadas de função e variáveis locais.

    Vantagens do uso de threads incluem:

    • Melhor Desempenho: Em sistemas com múltiplos núcleos, threads podem ser executadas em paralelo.
    • Responsividade: Aplicativos podem continuar respondendo ao usuário enquanto realizam operações em segundo plano.
    • Economia de Recursos: Criar threads é menos custoso do que criar processos completos.

    Desafios incluem:

    • Sincronização: Necessidade de mecanismos como mutexes e semáforos para evitar condições de corrida.
    • Concorrência: Garantir que o acesso a recursos compartilhados seja controlado.

    Threads podem ser gerenciadas a nível de usuário ou de kernel, cada uma com suas vantagens e desvantagens em termos de desempenho e complexidade.

    Exemplo Prático: Um navegador web pode usar múltiplas threads para carregar diferentes elementos de uma página simultaneamente, como imagens, scripts e textos.
