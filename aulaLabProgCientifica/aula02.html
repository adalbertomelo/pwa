Formulação de Algoritmos e sua Representação

Objetivos

  • Introduzir os conceitos fundamentais sobre algoritmos e suas propriedades.
  • Explorar as etapas de formulação de um algoritmo.
  • Apresentar diferentes formas de representação de algoritmos, como pseudocódigo e fluxogramas.
  • Aplicar a construção de algoritmos através de exemplos práticos.

1. Introdução aos Algoritmos

Um algoritmo é uma sequência finita de instruções bem definidas para resolver um problema ou executar uma tarefa.

Propriedades dos Algoritmos:

  • Finitude: O algoritmo deve terminar após um número finito de passos.
  • Clareza: Cada passo deve ser claro e sem ambiguidades.
  • Entradas e Saídas: O algoritmo deve ter zero ou mais entradas e pelo menos uma saída.
  • Eficiência: Deve ser eficiente em termos de tempo e espaço.
Exemplo: Algoritmo para somar dois números. 
Passo 1: Leia o primeiro número (A)
Passo 2: Leia o segundo número (B)
Passo 3: Some A e B (C = A + B)
Passo 4: Exiba o resultado (C)

2. Etapas de Formulação de Algoritmos

  • 1. Compreensão do Problema: Defina claramente o problema e identifique os dados de entrada e saída.
  • 2. Análise do Problema: Divida o problema em partes menores e escolha a abordagem mais eficiente.
  • 3. Desenvolvimento do Algoritmo: Defina os passos lógicos e as condições para chegar à solução.
  • 4. Teste e Validação: Teste o algoritmo para garantir sua precisão e eficiência.
Exemplo: Algoritmo para encontrar o maior número entre três valores. 
Passo 1: Leia os três números (A, B, C)
Passo 2: Se A for maior que B e A for maior que C, o maior é A.
Passo 3: Se B for maior que A e B for maior que C, o maior é B.
Passo 4: Caso contrário, o maior é C.

3. Representação de Algoritmos

Pseudocódigo: É uma maneira de escrever algoritmos de forma clara e intuitiva, sem a preocupação com a sintaxe de uma linguagem de programação específica.

Exemplo: Algoritmo de verificação de número par ou ímpar. 
Início
    Leia o número (N)
    Se N mod 2 = 0 então
        Exiba "Par"
    Senão
        Exiba "Ímpar"
Fim

Fluxogramas: Representam graficamente os passos de um algoritmo, utilizando formas geométricas como retângulos, losangos e paralelogramos.

4. Exemplos Práticos de Algoritmos

Exemplo 1: Algoritmo para calcular a média de uma lista de números. 
Passo 1: Leia a quantidade de números (n)
Passo 2: Para i de 1 até n, faça:
         Leia o número e adicione ao total.
Passo 3: Calcule a média (total / n)
Passo 4: Exiba a média.
Exemplo 2: Algoritmo para determinar se um número é primo. 
Passo 1: Leia o número (n)
Passo 2: Para i de 2 até √n, faça:
         Se n for divisível por i, exiba "Não é primo" e termine.
Passo 3: Se nenhum divisor for encontrado, exiba "É primo".
Exemplo 3: Algoritmo de ordenação por seleção. 
Passo 1: Para i de 0 até n-1 faça:
         a) Encontre o menor elemento da lista não ordenada.
         b) Troque-o com o primeiro elemento não ordenado.
Passo 2: Repita até que toda a lista esteja ordenada.

5. Atividades Práticas

Atividade 1: Crie um algoritmo para calcular o fatorial de um número.
Atividade 2: Desenvolva um algoritmo para ordenar uma lista de números usando o método da bolha (bubble sort).
Atividade 3: Escreva um algoritmo que, dado um número inteiro, determine a soma de todos os números ímpares entre 1 e o número fornecido.
Atividade 4: Crie um fluxograma que represente o algoritmo de busca binária para encontrar um número em uma lista ordenada.

6. Conclusão

Os algoritmos são a base da programação e são fundamentais para resolver problemas de forma eficiente. A prática na formulação e representação de algoritmos ajuda a desenvolver o raciocínio lógico e a capacidade de resolver problemas complexos. Durante esta aula, vimos diferentes formas de representar e formular algoritmos, desde pseudocódigo até fluxogramas, e aplicamos esses conceitos em exemplos práticos.
