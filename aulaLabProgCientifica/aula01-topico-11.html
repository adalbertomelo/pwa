4.3. Memória Cache

  • Conceito de Memória Cache

    A Memória Cache é uma memória de alta velocidade que atua como intermediária entre a CPU e a memória principal (RAM). Seu objetivo é reduzir o tempo de acesso aos dados frequentemente utilizados, melhorando o desempenho geral do sistema.

    Características:

    • Velocidade maior que a da memória RAM, mas menor capacidade.
    • Armazena cópias dos dados mais acessados da memória principal.
    • Transparente para o usuário e para os programas.

    Tipos de Cache:

    • Cache L1: Integrado ao processador, com tamanho pequeno e alta velocidade.
    • Cache L2: Também pode ser integrado ou externo ao processador, maior que o L1 e um pouco mais lento.
    • Cache L3: Presente em processadores mais modernos, ainda maior e mais lento que o L2.

    Funcionamento:

    • Quando a CPU precisa acessar dados, ela verifica primeiro o cache.
    • Se os dados estiverem no cache (cache hit), o acesso é rápido.
    • Se não estiverem (cache miss), os dados são buscados na memória RAM e uma cópia é armazenada no cache.

    O uso eficiente da memória cache é essencial para minimizar a discrepância de velocidade entre a CPU e a memória principal.

    Exemplo Prático: Ao executar um loop que acessa repetidamente a mesma variável, essa variável será armazenada no cache, acelerando as operações subsequentes.

  • Políticas de Substituição

    Quando o cache está cheio e novos dados precisam ser armazenados, o sistema precisa decidir quais dados existentes serão removidos. As Políticas de Substituição determinam como essa escolha é feita.

    Principais Políticas:

    • FIFO (First-In, First-Out): Remove o bloco mais antigo no cache.
    • LRU (Least Recently Used): Remove o bloco que não é usado há mais tempo.
    • LFU (Least Frequently Used): Remove o bloco menos acessado no total.
    • Random Replacement: Remove um bloco aleatoriamente.

    Considerações:

    • A escolha da política afeta o desempenho do cache.
    • LRU é comumente usada por oferecer um bom equilíbrio entre complexidade e eficiência.
    • Implementações eficientes de LRU podem ser complexas em hardware.

    O objetivo das políticas de substituição é minimizar a taxa de cache miss, mantendo os dados mais relevantes no cache.

    Exemplo Prático: Em um processador que utiliza LRU, se um conjunto de dados é acessado com frequência, eles permanecerão no cache, enquanto dados menos utilizados serão substituídos.
