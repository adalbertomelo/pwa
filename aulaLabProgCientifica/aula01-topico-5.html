2. Sistemas Operacionais

2.1. Funções Principais

  • Gerenciamento de Processos

    O sistema operacional é responsável por criar, agendar e finalizar processos. Ele gerencia a execução de múltiplos programas simultaneamente, garantindo que cada processo receba tempo suficiente de CPU para funcionar corretamente. Isso inclui a troca de contexto entre processos, sincronização e comunicação entre processos. O sistema operacional também lida com threads, que são subunidades de processos, permitindo multitarefa e melhor utilização dos recursos do sistema.

    Exemplo Prático: Quando você está ouvindo música enquanto navega na internet, o sistema operacional gerencia ambos os processos para que funcionem simultaneamente sem interferências.

  • Gerenciamento de Memória

    Este componente gerencia a alocação e liberação de memória para processos em execução. Ele mantém o controle de quais partes da memória estão sendo utilizadas e por quais processos, além de decidir quais processos terão memória alocada ou liberada. O gerenciamento de memória inclui técnicas como paginação, segmentação e memória virtual, que permitem que o sistema operacional forneça a ilusão de uma grande memória contínua, mesmo que a memória física seja limitada.

    Exemplo Prático: Ao abrir vários aplicativos em um computador com memória limitada, o sistema operacional pode usar memória virtual para continuar operando, movendo dados menos usados para o disco rígido temporariamente.

  • Gerenciamento de Armazenamento

    O sistema operacional gerencia a leitura e escrita de dados em dispositivos de armazenamento, como discos rígidos e SSDs. Ele organiza os dados em estruturas chamadas sistemas de arquivos, permitindo que os usuários criem, modifiquem e excluam arquivos e diretórios. Além disso, lida com a segurança e permissões de acesso aos arquivos, garantindo a integridade e confidencialidade dos dados armazenados.

    Exemplo Prático: Quando você salva um documento no computador, o sistema operacional determina onde os dados serão armazenados no disco e atualiza o sistema de arquivos adequadamente.

  • Gerenciamento de Dispositivos

    Controla a interação com dispositivos de hardware periféricos. O sistema operacional utiliza drivers para se comunicar com dispositivos como impressoras, placas de som, placas de vídeo e dispositivos USB. Isso permite que os aplicativos acessem hardware sem precisar conhecer detalhes específicos de cada dispositivo, promovendo a compatibilidade e facilitando a adição de novos dispositivos ao sistema.

    Exemplo Prático: Ao conectar um novo mouse ao computador, o sistema operacional carrega o driver apropriado para que o dispositivo funcione corretamente sem a necessidade de configuração manual.

  • Interface com o Usuário

    Fornece meios para que os usuários interajam com o sistema operacional e executem tarefas. Isso pode ser feito através de uma Interface de Linha de Comando (CLI), onde comandos são inseridos via texto, ou uma Interface Gráfica de Usuário (GUI), que utiliza elementos visuais como janelas, ícones e menus. A interface com o usuário é essencial para a usabilidade do sistema, permitindo que usuários de diferentes níveis de experiência utilizem o computador de forma eficiente.

    Exemplo Prático: O ambiente de trabalho do Windows, com sua área de trabalho, menu Iniciar e barra de tarefas, é uma interface gráfica que permite aos usuários acessar facilmente aplicativos e configurações.
