5. Sistemas de Arquivos

5.1 Organização de Arquivos

  • Estrutura de Diretórios

    A estrutura de diretórios organiza os arquivos de um sistema operacional de forma hierárquica. Isso facilita a localização, armazenamento e gerenciamento de arquivos. Os tipos comuns de estruturas de diretórios incluem:

    • Estrutura em Árvore: Diretórios são organizados em forma hierárquica, com um único diretório raiz.
    • Estrutura em Grafo Acíclico: Permite que arquivos ou diretórios tenham múltiplos pais, através de links ou atalhos.

    A organização adequada da estrutura de diretórios é essencial para a eficiência do sistema e facilidade de uso pelos usuários.

    Exemplo Prático: Em sistemas Unix/Linux, o diretório raiz é representado por "/", e a hierarquia se estende para diretórios como "/home", "/usr", "/bin", etc.

5.2 Métodos de Acesso

  • Acesso Sequencial

    No Acesso Sequencial, os dados são acessados em uma ordem específica, geralmente do início ao fim. É o método mais simples de acesso a arquivos.

    Características:

    • Os dados são lidos ou escritos em sequência.
    • A posição atual é atualizada após cada operação.
    • A busca por um dado específico pode ser lenta, pois requer a leitura dos dados anteriores.

    Exemplo Prático: Leitura de um arquivo de texto linha por linha.

  • Acesso Direto (Aleatório)

    O Acesso Direto permite que os dados sejam lidos ou escritos em qualquer ordem, sem a necessidade de percorrer sequencialmente.

    Características:

    • Os arquivos são divididos em registros ou blocos numerados.
    • É possível saltar diretamente para qualquer posição no arquivo.
    • Mais eficiente para operações que requerem acesso frequente a posições específicas.

    Exemplo Prático: Bancos de dados que precisam acessar registros específicos rapidamente.

Sistemas de Arquivos Comuns

  • FAT32

    O FAT32 (File Allocation Table 32) é um sistema de arquivos introduzido pela Microsoft. É uma extensão do FAT16 e suporta tamanhos de partição e arquivo maiores.

    Características:

    • Compatibilidade ampla com diferentes sistemas operacionais.
    • Limite de tamanho de arquivo de até 4 GB.
    • Sem suporte a permissões de arquivos ou recursos avançados.

    Vantagens:

    • Ideal para dispositivos de armazenamento removíveis como pen drives e cartões SD.
    • Simples de implementar e gerenciar.

    Desvantagens:

    • Limitações de tamanho de arquivo e partição.
    • Menor segurança e falta de criptografia nativa.

    Exemplo Prático: Ao formatar um pen drive para transferir arquivos entre computadores com diferentes sistemas operacionais, o FAT32 é uma opção compatível com a maioria deles.

  • NTFS

    O NTFS (New Technology File System) é um sistema de arquivos desenvolvido pela Microsoft para substituir o FAT32, oferecendo recursos avançados.

    Características:

    • Suporte a arquivos e partições de grande tamanho (teoricamente até 16 EB).
    • Permissões e controle de acesso avançados.
    • Recuperação de falhas com registro de transações.
    • Criptografia de arquivos (EFS - Encrypting File System).

    Vantagens:

    • Maior segurança e confiabilidade.
    • Melhor desempenho com arquivos grandes.

    Desvantagens:

    • Compatibilidade limitada com sistemas operacionais não Windows.
    • Maior complexidade de gerenciamento.

    Exemplo Prático: O NTFS é o sistema de arquivos padrão para discos rígidos internos em computadores Windows, oferecendo recursos necessários para ambientes corporativos.

  • ext4

    O ext4 (Fourth Extended File System) é um sistema de arquivos usado principalmente em sistemas Linux, sucedendo os sistemas ext2 e ext3.

    Características:

    • Suporte a volumes de até 1 EB e arquivos de até 16 TB.
    • Alocação tardia (delayed allocation) para melhorar desempenho.
    • Compatibilidade com ext2 e ext3.
    • Journaling para recuperação de falhas.

    Vantagens:

    • Alto desempenho e estabilidade.
    • Recursos avançados como extents e checagem de integridade.

    Desvantagens:

    • Não é nativamente compatível com Windows ou macOS.
    • Recuperação de dados mais complexa em caso de falhas graves.

    Exemplo Prático: A maioria das distribuições Linux utiliza o ext4 como sistema de arquivos padrão para a partição raiz, devido ao seu equilíbrio entre desempenho e recursos.
