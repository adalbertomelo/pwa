7. Noções de Redes de Computadores

7.1. Modelos de Rede

  • Modelo OSI

    O Modelo OSI (Open Systems Interconnection) é um modelo de referência com sete camadas que padroniza as funções de uma rede de comunicação.

    As Sete Camadas:

    1. Física: Transmissão de bits brutos pelo meio físico.
    2. Enlace de Dados: Transferência confiável de dados entre dois nós adjacentes.
    3. Rede: Determinação de rotas e encaminhamento de pacotes.
    4. Transporte: Comunicação confiável entre hosts finais.
    5. Sessão: Gerenciamento de sessões entre aplicações.
    6. Apresentação: Tradução, compressão e criptografia de dados.
    7. Aplicação: Serviços de rede para aplicativos.

    O modelo OSI é fundamental para entender como as diferentes camadas interagem e permitem a comunicação em redes complexas.

    Exemplo Prático: Protocolos como Ethernet operam na camada de Enlace de Dados, enquanto o HTTP opera na camada de Aplicação.

  • Modelo TCP/IP

    O Modelo TCP/IP é um modelo de arquitetura de protocolos em quatro camadas, desenvolvido para a internet.

    As Quatro Camadas:

    1. Camada de Acesso à Rede: Combina funções das camadas Física e de Enlace de Dados do modelo OSI.
    2. Camada Internet: Equivalente à camada de Rede do OSI; responsável pelo roteamento (exemplo: protocolo IP).
    3. Camada de Transporte: Proporciona comunicação entre processos (exemplos: TCP, UDP).
    4. Camada de Aplicação: Combina as camadas de Sessão, Apresentação e Aplicação do OSI; fornece serviços para aplicativos (exemplos: HTTP, FTP, SMTP).

    O modelo TCP/IP é a base da internet atual, permitindo a interoperabilidade entre diferentes sistemas e redes.

    Exemplo Prático: Quando você acessa um site, seu navegador utiliza protocolos da camada de Aplicação (HTTP/HTTPS) sobre TCP/IP para se comunicar com o servidor.

7.2. Protocolos Básicos

  • Protocolo IP (Internet Protocol)

    O Protocolo IP é o protocolo principal da camada Internet do modelo TCP/IP. Ele é responsável pelo endereçamento e roteamento dos pacotes de dados entre redes.

    Características:

    • Protocolo não orientado à conexão.
    • Protocolo sem garantia de entrega (melhor esforço).
    • Utiliza endereços IP para identificar dispositivos na rede.

    Versões:

    • IPv4: Utiliza endereços de 32 bits.
    • IPv6: Utiliza endereços de 128 bits, expandindo o espaço de endereçamento.

    Exemplo Prático: O IP roteia pacotes de dados da sua máquina até o servidor de destino na internet.

  • Protocolo TCP (Transmission Control Protocol)

    O TCP é um protocolo da camada de Transporte que fornece comunicação confiável, orientada à conexão entre aplicações.

    Características:

    • Estabelece uma conexão antes da transferência de dados (handshake de três vias).
    • Garante a entrega dos dados na ordem correta.
    • Controle de fluxo e congestionamento.

    Exemplo Prático: Ao enviar um e-mail ou acessar um site, o TCP garante que todos os dados sejam entregues corretamente.

  • Protocolo UDP (User Datagram Protocol)

    O UDP é um protocolo da camada de Transporte que oferece comunicação não confiável, não orientada à conexão.

    Características:

    • Não estabelece conexão antes da transferência de dados.
    • Menor overhead, permitindo transferências mais rápidas.
    • Não garante a entrega ou a ordem dos dados.

    Exemplo Prático: Aplicações como streaming de vídeo ao vivo ou jogos online utilizam UDP para minimizar latência.
